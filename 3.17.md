# 3.17 更新说明

## 1. 这次改了什么

这次针对 `Common Carotid Artery Data Set / ARTERY_TRANSVERSAL` 这份数据，重新梳理并修改了整套流程。

核心变化：

- 保留原始的分割训练链路不变
  - `cubs_v1 / cubs_v2 / cca` 还是走原来的 `prepare_data.py -> train.py -> predict.py`
- 新增一套专门给 `cca_artifact` 用的“双模型”流程
  - 模型 1：整图定位模型
  - 模型 2：patch 真假分类模型
- 不再把这份数据强行当成标准三类分割数据来用
- 新的 `cca_artifact` 数据准备逻辑会生成两个子数据集，而不是一套 `images + masks`


## 2. 为什么要改

之前一版的做法是把这份数据改造成“背景 / 伪影 / 真实”三类 mask，再走分割训练。

这个方案工程上能跑，但不完全符合数据集原始设计，问题在于：

- `img + txt` 提供的是整图中的目标框位置
- `positive + negative` 提供的是局部 patch 的真假样本
- 这两部分监督本来就是两种不同任务
- `positive / negative` 并不是天然和某张整图一一对齐的像素级 mask

所以更合理的理解方式应该是：

1. 用整图和框标注做定位
2. 用 patch 做真假分类


## 3. 数据集原始组织方式

目录：

- `ARTERY_TRANSVERSAL/Ultrasonix_train/img`
  - 训练整图
- `ARTERY_TRANSVERSAL/Ultrasonix_train/txt`
  - 每张整图的目标框标注
- `ARTERY_TRANSVERSAL/Ultrasonix_train/positive`
  - 正样本 patch，表示真实血管横断面
- `ARTERY_TRANSVERSAL/Ultrasonix_train/negative`
  - 负样本 patch，表示伪影 / 假目标
- `ARTERY_TRANSVERSAL/Ultrasonix_test`
  - Ultrasonix 设备测试整图和框
- `ARTERY_TRANSVERSAL/Toshiba_test`
  - Toshiba 设备测试整图和框

这说明它更像：

- 一个“整图定位数据集”
- 再加一个“局部真假分类数据集”


## 4. 新的数据准备逻辑

现在执行：

```bash
python data/prepare_data.py \
  --datasets cca_artifact \
  --cca_artifact_root "data/extracted/Common Carotid Artery Data Set" \
  --out_root data/processed/cca_artifact_dataset
```

不会再生成旧版那种三类 mask 数据集，而是会生成两个子数据集：

### 4.1 定位数据集

路径：

- `data/processed/cca_artifact_dataset/artifact_localization`

内容：

- `images/`
- `metadata_all.csv`
- `splits/train.csv`
- `splits/val.csv`
- `splits/test.csv`
- `dataset_summary.json`

监督形式：

- 每张整图对应一个 bbox
- 模型学习输出血管横断面的框位置

划分原则：

- `Ultrasonix_train` 作为 train/val 来源
- `Ultrasonix_test + Toshiba_test` 作为 test

### 4.2 patch 分类数据集

路径：

- `data/processed/cca_artifact_dataset/artifact_patch_cls`

内容：

- `images/`
- `metadata_all.csv`
- `splits/train.csv`
- `splits/val.csv`
- `splits/test.csv`
- `dataset_summary.json`

监督形式：

- `positive -> real_artery`
- `negative -> artifact_fake`


## 5. 两个模型分别做什么

### 5.1 定位模型

脚本：

- `train_artifact_localizer.py`
- `run_artifact_localizer.sh`

输入：

- 一整张超声图

输出：

- 一个 bbox

任务：

- 找到“血管横断面在哪里”

工作原理：

- 使用 DINOv3 backbone 提特征
- 对全局特征做回归
- 直接输出归一化后的 `cx, cy, w, h`


### 5.2 patch 分类模型

脚本：

- `train_artifact_patch_cls.py`
- `run_artifact_patch_cls.sh`

输入：

- 一个裁剪 patch

输出：

- 二分类结果
  - `artifact_fake`
  - `real_artery`

任务：

- 判断局部候选区域是真是假

工作原理：

- 使用 DINOv3 backbone 提特征
- 通过分类头输出两类 logits


## 6. 两阶段推理怎么工作

脚本：

- `predict_artifact_pipeline.py`
- `predict_artifact.sh`

流程：

1. 用定位模型在整图上预测 bbox
2. 从预测框中裁剪 patch
3. 用 patch 分类模型判断这个 patch 是真还是伪影
4. 生成输出

现在的输出策略：

- 如果判定为 `artifact_fake`
  - 输出 mask 中对应框区域填灰色
- 如果判定为 `real_artery`
  - 输出 mask 维持黑色

也就是说当前 `predict_artifact.sh` 的 mask 输出重点是“伪影提示图”，不是“真实区域白色分割图”。

## 9. 兼容性说明

这次改动保留了原始分割链路：

- `train.py`
- `predict.py`
- `run.sh`
- `run_multi.sh`

这些仍然用于最开始的分割任务，不受新的 artifact 双模型流程影响。

也就是说现在项目里是两套并行方案：

1. 原始分割方案
2. 新的 `cca_artifact` 双模型方案


## 10. 目前已知限制

### 10.1 定位模型不是完整检测器

现在的定位模型本质上是“单目标框回归”，不是 YOLO 那种通用多目标检测器。

但对这份数据来说，这样更贴近实际，因为每张图主要就是一个目标区域。

### 10.2 patch 分类和定位是串行的

当前推理是：

- 先定位
- 再分类

优点是更符合数据原始设计；
缺点是如果第一步框偏了，第二步分类也会受影响。

### 10.3 当前输出 mask 更偏“伪影提示”

现在 `predict_artifact.sh` 输出的 mask 重点标伪影，不再把真实区域单独画成白色。

如果后续需要，也可以再扩展成：

- 伪影 = 灰色
- 真实 = 白色
- 背景 = 黑色


## 11. 建议

如果目标是“做一套真正符合这份数据原意的模型”，当前新方案比旧的三类弱监督分割更合适。

建议优先关注：

1. 定位模型框是否稳定
2. patch 分类模型真假区分能力
3. 两阶段串联后的最终误判情况

如果后面效果不理想，优先排查：

- 定位框偏差
- patch 分类器是否过拟合
- train / val / test 分布差异
